SECURITY GOVERNANCE RULES
Refresh Tokens
Never store raw refresh tokens.
Store:
refresh_token_idrefresh_token_hash
only.

Password Reset Tokens
Never store raw reset tokens.
Store:
reset_token_hash
only.

Email Verification Tokens
Never store raw verification tokens.
Store:
verification_token_hash
only.

Trusted Devices
Version 1 supports:
Remember DeviceTrusted Device TrackingActive Device Listing

Session Revocation
Revoked sessions must capture:
revocation_reason
Examples:
Manual LogoutPassword ChangeAdmin RevokedSecurity Lock

Password Change Rule
When password changes:
All active sessions→ Revoked
except current session if configured by application settings.

Archival handled by application maintenance jobs.
