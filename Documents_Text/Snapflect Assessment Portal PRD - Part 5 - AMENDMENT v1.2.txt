A5.21 QUESTION ANSWER NORMALIZATION RULES
Purpose
Prevent answer duplication and maintain normalized question structures.

Multiple Choice Questions
Question Types:
Single Choice
Multiple Choice
True/False

Storage Strategy
Correct answers shall be stored in:
question_options
using:
is_correct

Example
Option A
is_correct = 0
Option B
is_correct = 1
Option C
is_correct = 0
Option D
is_correct = 0

Rule
No record shall be created in:
question_answers
for these question types.

Non-MCQ Questions
Question Types:
Fill In Blank
Descriptive
Code Snippet
Scenario Based
Case Study

Storage Strategy
Correct answers shall be stored in:
question_answers

Example
Question:
What is Dependency Injection?
Answer:
A design pattern implementing inversion of control.

Rule
question_options is optional for these question types.
question_answers becomes the authoritative answer source.

Data Integrity Rule
question_options
and
question_answers
must never duplicate answer definitions.
Only one answer source shall be authoritative for a given question.

A5.23 ERD GENERATION CONSTRAINTS
The ERD must comply with all architecture rules.

Mandatory Requirements
All entities must have a clear ownership boundary.
All entities must support auditability.
All entities must support soft delete where applicable.
All relationships must be explicitly defined.
All foreign keys must be represented.
All many-to-many relationships must use mapping tables.

Explicit Constraints
Do NOT create:
question_usage_analysis
table.
Do NOT duplicate answer storage.
Do NOT create circular relationships.
Do NOT allow direct scoring against mutable master data.
