SYSTEM ROLES

PERMISSION NAMING STANDARD
Format:
module.action
Examples:
technology.createtechnology.edittechnology.publishassessment.createassessment.editassessment.publishquestion.createquestion.editassignment.createreview.approvereport.export

ROLE GOVERNANCE RULES
System Roles
System roles:
SUPER_ADMINASSESSMENT_ADMINUSER
cannot be deleted.

Custom Roles
Custom roles may be:
CreatedEditedDeactivated

Active Assignments
Roles having active user assignments:
user_roles.is_active = 1
cannot be deleted.
They must be deactivated instead.

PERMISSION GOVERNANCE RULES
Permissions are:
Soft Deletable
but:
permission_code
must never be reused.

ROLE ASSIGNMENT HISTORY RULES
Historical assignments are preserved.
Removing a role from a user shall:
Set is_active = 0Populate effective_to
No records are physically deleted.
