This document establishes the implementation assumptions and seed data strategy required before generating MySQL schema scripts.
Objectives:
Eliminate ambiguity during schema creation
Standardize authentication and authorization design
Define initial system data
Establish migration sequencing
Define environment initialization requirements

2. AUTHENTICATION STRATEGY
Login Method

Email Address

User Registration
Supported:
Self Registration
and
Admin User Creation

Authentication Mechanism
JWT Access TokenJWT Refresh Token

Token Storage
Application Layer Responsibility
Database stores:
Session MetadataToken Reference IDs
Database shall NOT store:
Plain JWT Tokens

Password Storage
Algorithm:

Minimum Cost Factor:
12

Prohibited
MD5SHA1Plain Text Passwords

3. ACCOUNT STATUS MODEL
Supported Status Values
ActivePending VerificationLockedSuspendedDeleted

Business Rules
Active
Can login and access application.

Pending Verification
Email verification required.

Locked
Temporary lock due to security events.

Suspended
Administrative action.

Deleted
Soft deleted account.
Cannot login.
