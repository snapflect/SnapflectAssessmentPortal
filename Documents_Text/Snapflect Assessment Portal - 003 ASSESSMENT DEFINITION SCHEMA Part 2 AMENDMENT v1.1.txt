1. PURPOSE OF AMENDMENT
This amendment introduces a critical version-integrity enhancement to the Assessment Definition architecture.
The original Part 2 design stored:
section_question_banks    ↓question_bank_id
This approach does not guarantee historical reconstruction of published assessment content when Question Banks evolve over time.
The amendment updates the architecture to reference:
question_bank_version_id
instead of:
question_bank_id
thereby preserving complete content version integrity.

2. PROBLEM IDENTIFIED
Original Architecture
Assessment Version        ↓Section Version        ↓Section Question Banks        ↓Question Bank

Example Scenario
Assessment v1        ↓Section v1        ↓Question Bank v1
Later:
Question Bank v2Question Bank v3
are created.
Because the mapping references only:
question_bank_id
the platform cannot reliably determine which Question Bank version belonged to the published Assessment Version.

Impact
Potential issues:
Historical Reconstruction FailureAssessment Replay InaccuracyReview InconsistenciesReporting InaccuraciesAudit Traceability Gaps

3. APPROVED SOLUTION
New Architecture
Assessment Version        ↓Section Version        ↓Section Question Banks        ↓Question Bank Version

Final Content Version Hierarchy
Technology Version        ↓Assessment Version        ↓Section Version        ↓Question Bank Version        ↓Question Version
This architecture aligns with the approved Full Snapshot Versioning strategy.
