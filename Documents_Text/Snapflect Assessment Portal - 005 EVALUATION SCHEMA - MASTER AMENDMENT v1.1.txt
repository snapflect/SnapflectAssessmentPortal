2. EVALUATION_RESULTS ENHANCEMENT
2.1 Current Result Tracking
New Column
is_current_version TINYINT(1)NOTNULLDEFAULT1

Purpose
Identifies the active evaluation result version.

Example
Result Version 1    ↓ supersededResult Version 2    ↓ supersededResult Version 3
Only:
Result Version 3
contains:
is_current_version = 1

Benefits
Supports:
Fast Result RetrievalAppealsRe-EvaluationCertification Processing
without recursive traversal.

2.2 Result Version Governance
When a new result version is created:
Previous Version
must be updated:
is_current_version = 0

Only one current version may exist for:
assessment_attempt_id

7. GOVERNANCE RULES
Current Result Rule
For a given:
assessment_attempt_id
only one evaluation result may have:
is_current_version = 1

Publication Rule
When:
publication_status = Published
the following should be populated:
publication_datepublished_by

Review SLA Rule
When:
assignment_status = Accepted
the reviewer should populate:
review_started_date
before review completion.

Scheduled Evaluation Rule
When:
scheduled_date
exists:
evaluation_jobs
must not begin execution before that timestamp.

Rule Assignment Effective Dating Rule
Rule assignments are active only when:
current_timestamp >= effective_from
and
current_timestamp < effective_to
when an end date exists.
