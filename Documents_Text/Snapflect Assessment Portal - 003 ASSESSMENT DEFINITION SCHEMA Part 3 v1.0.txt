QUESTION_BANKS
Purpose
Represents reusable question bank libraries.
Question Banks are independent content repositories and are not directly owned by:
TechnologyAssessmentSection
Instead, Question Banks are attached to Section Versions through:
section_question_banks
This architecture enables content reuse.

CREATETABLE question_banks(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    question_bank_code VARCHAR(100) NOTNULL,    ownership_type VARCHAR(50)NOTNULLDEFAULT'Custom',    status VARCHAR(50)NOTNULLDEFAULT'Draft',    created_by BIGINT UNSIGNED NOTNULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),    modified_by BIGINT UNSIGNED NULL,    modified_date DATETIME(3) NULL,    is_active TINYINT(1)NOTNULLDEFAULT1,    is_deleted TINYINT(1)NOTNULLDEFAULT0,    deleted_by BIGINT UNSIGNED NULL,    deleted_date DATETIME(3) NULL,PRIMARYKEY (id),CONSTRAINT ux_question_banks_uuidUNIQUE (uuid),CONSTRAINT ux_question_banks_codeUNIQUE (question_bank_code))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

QUESTION_BANK_VERSIONS
Purpose
Stores complete historical snapshots of Question Bank definitions.
Uses Full Snapshot Versioning Architecture.

CREATETABLE question_bank_versions(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    question_bank_id BIGINT UNSIGNED NOTNULL,    version_number INTNOTNULL,    version_status VARCHAR(50)NOTNULLDEFAULT'Draft',    question_bank_name VARCHAR(255) NOTNULL,    question_bank_category VARCHAR(100) NOTNULL,    difficulty_level VARCHAR(50)NOTNULLDEFAULT'Medium',    selection_strategy VARCHAR(50)NOTNULLDEFAULT'AllQuestions',    question_count INTNULL,    randomize_question_order TINYINT(1)NOTNULLDEFAULT0,    estimated_duration_minutes INTNULL,    passing_percentage DECIMAL(5,2) NULL,    instructions TEXT NULL,    is_visible TINYINT(1)NOTNULLDEFAULT1,    created_by BIGINT UNSIGNED NOTNULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),PRIMARYKEY (id),CONSTRAINT ux_question_bank_versions_uuidUNIQUE (uuid),CONSTRAINT ux_question_bank_versions_versionUNIQUE (question_bank_id, version_number))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

RANDOMIZATION STRATEGY
Supported through:
randomize_question_order
Values:
0 = Fixed Order1 = Random Order

QUESTION COUNT RULES
When:
selection_strategy = RandomSelection
then:
question_count
must contain a valid value.

When:
selection_strategy = AllQuestions
then:
question_count
may remain NULL.

RUNTIME VALIDATION RULE
Application services must enforce:
question_count<=available active questions
for the Question Bank Version.

VISIBILITY RULES
Question Banks with:
is_visible = 0
must not appear in administrative selection screens unless explicitly requested.
