NOTIFICATION_QUEUE
Purpose
Stores notification messages awaiting delivery.
Notifications are created from immutable template versions and store content snapshots to guarantee historical reproducibility.

Processing Lifecycle
Business Event        ↓Notification Queue        ↓Delivery Worker        ↓Provider        ↓Delivery Result

CREATETABLE notification_queue(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    notification_template_version_id BIGINT UNSIGNED NOTNULL,    notification_channel_id BIGINT UNSIGNED NOTNULL,    recipient_user_id BIGINT UNSIGNED NULL,    recipient_address VARCHAR(500) NOTNULL,    subject_snapshot TEXT NULL,    body_snapshot LONGTEXT NOTNULL,    priority VARCHAR(50)NOTNULLDEFAULT'Normal',    correlation_id CHAR(36) NULL,    idempotency_key VARCHAR(255) NOTNULL,    retry_count INTNOTNULLDEFAULT0,    max_retry_count INTNOTNULLDEFAULT3,    scheduled_date DATETIME(3) NULL,    processed_date DATETIME(3) NULL,    status VARCHAR(50)NOTNULLDEFAULT'Pending',    metadata_json JSON NULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),PRIMARYKEY (id),CONSTRAINT ux_notification_queue_uuidUNIQUE (uuid),CONSTRAINT ux_notification_queue_idempotencyUNIQUE (idempotency_key))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

NOTIFICATION_DELIVERY_LOGS
Purpose
Stores individual delivery attempts.
Each retry generates a new log entry.

Example
