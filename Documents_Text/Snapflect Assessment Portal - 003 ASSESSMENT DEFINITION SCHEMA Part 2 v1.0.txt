SECTIONS
Purpose
Represents logical assessment groupings.
Examples:
BasicsIntermediate ConceptsAdvanced TopicsArchitecturePerformance
A Section belongs to exactly one Assessment.

SECTION_VERSIONS
Purpose
Stores complete historical snapshots of Section definitions.
Uses Full Snapshot Versioning Architecture.

CREATETABLE section_versions(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    section_id BIGINT UNSIGNED NOTNULL,    version_number INTNOTNULL,    version_status VARCHAR(50)NOTNULLDEFAULT'Draft',    section_name VARCHAR(255) NOTNULL,    display_order INTNOTNULLDEFAULT0,    section_type VARCHAR(50)NOTNULLDEFAULT'Mandatory',    instructions TEXT NULL,    estimated_duration_minutes INTNULL,    passing_percentage DECIMAL(5,2) NULL,    navigation_mode VARCHAR(50)NOTNULLDEFAULT'Sequential',    is_visible TINYINT(1)NOTNULLDEFAULT1,    max_attempts INTNULL,    created_by BIGINT UNSIGNED NOTNULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),PRIMARYKEY (id),CONSTRAINT ux_section_versions_uuidUNIQUE (uuid),CONSTRAINT ux_section_versions_versionUNIQUE (section_id, version_number))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

SECTION_QUESTION_BANKS
Purpose
Maps Question Banks to a specific Section Version.
This preserves historical version integrity.

Architecture
Section Version        ↓Section Question Banks        ↓Question Bank

CREATETABLE section_question_banks(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    section_version_id BIGINT UNSIGNED NOTNULL,    question_bank_id BIGINT UNSIGNED NOTNULL,    display_order INTNOTNULLDEFAULT0,    is_mandatory TINYINT(1)NOTNULLDEFAULT1,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),PRIMARYKEY (id),CONSTRAINT ux_section_question_banks_uuidUNIQUE (uuid),CONSTRAINT ux_section_question_bank_mappingUNIQUE (            section_version_id,            question_bank_id        ))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;
