1. PURPOSE
This document defines the database architecture standards, naming conventions, normalization rules, indexing strategy, audit strategy, versioning strategy, and implementation rules for the Snapflect Assessment Portal.
All database objects must comply with this standard.

2. DATABASE PLATFORM
Database Engine:
MySQL 8+
Storage Engine:
InnoDB
Character Set:
utf8mb4
Collation:
utf8mb4_unicode_ci
Timezone:
UTC

3. DATABASE DESIGN PRINCIPLES
The database shall follow:
Third Normal Form (3NF)
Referential Integrity
Soft Delete Strategy
Auditability
Versioning
Snapshot Isolation
Extensibility
Performance Optimization

4. PRIMARY KEY STANDARD
All business entities shall use:
id BIGINT UNSIGNED AUTO_INCREMENT PRIMARYKEY

Reason
Supports:
Large datasets
Fast joins
Future scalability
Cursor AI consistency

5. UUID STANDARD
All major business entities shall additionally contain:
uuid CHAR(36) NOTNULL

Unique Constraint
UNIQUE(uuid)

Purpose
External API references
Public URLs
Import/Export Safety
Future Distributed Systems

6. FOREIGN KEY STANDARD
Foreign keys shall use:
BIGINT UNSIGNED

Example
user_id BIGINT UNSIGNED NOTNULL

Naming Convention
<entity>_id
Examples:
user_idassessment_idtechnology_idquestion_id

7. TABLE NAMING STANDARD
All tables shall use:
snake_case
plural names

Examples
usersrolespermissionsassessment_attemptsquestion_bank_questions

Prohibited
UsertblUsersAssessmentAttempt

8. COLUMN NAMING STANDARD
All columns shall use:
snake_case

Examples
created_datemodified_dateattempt_numberassessment_name

9. BOOLEAN STANDARD
Use:
TINYINT(1)

Examples
is_activeis_deletedis_current_versionis_correct

Do Not Use
BIT

10. ENUM STRATEGY
Do not use MySQL ENUM.
Use:
VARCHAR(50)

Examples
statusreview_statusapproval_statusassignment_type

Benefits
Easier upgrades
No schema changes for new values
Better maintainability

11. DATE & TIME STANDARD
Use:
DATETIME
for business events.

Examples
created_datesubmitted_datereviewed_date

Store All Dates
UTC

12. AUDIT FIELD STANDARD
Applicable to all business entities.

Standard Columns
created_by BIGINT UNSIGNED NULLcreated_date DATETIME NOTNULLmodified_by BIGINT UNSIGNED NULLmodified_date DATETIME NULL

Purpose
Traceability
Compliance
Auditability

13. SOFT DELETE STANDARD
Applicable to designated entities.

Standard Columns
is_deleted TINYINT(1) NOTNULLDEFAULT0deleted_by BIGINT UNSIGNED NULLdeleted_date DATETIME NULL

Rule
Physical deletes are prohibited.

14. ACTIVE STATUS STANDARD
Entities supporting activation shall contain:
is_active TINYINT(1) NOTNULLDEFAULT1

Purpose
Enable/Disable without deletion.

25. CASCADE RULES
Default:
RESTRICT

Avoid
ONDELETECASCADE
for business entities.

Reason
Protect historical data.

26. LARGE TEXT STANDARD
Use:
TEXT
for:
Descriptions
Comments
Instructions

Use:
LONGTEXT
for:
Snapshots
Large JSON payloads

27. JSON STORAGE STANDARD
MySQL JSON may be used where appropriate.
Preferred:
JSON
when querying is expected.
Use:
LONGTEXT
for immutable snapshot archives.

28. PERFORMANCE GUIDELINES
Avoid:
Excessive joins
Repeated subqueries
Redundant tables
Duplicate data

Prefer:

Caching
Derived Services
Pagination

Do NOT:
Use MySQL ENUM
Use physical deletes
Store passwords in plain text
Store JWT tokens in plain text
Create redundant analytics tables
Duplicate answer definitions
Create circular relationships
Bypass snapshot architecture
