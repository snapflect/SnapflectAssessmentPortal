A4.20 DATA GOVERNANCE REQUIREMENTS
Assignment reporting must be auditable.
Progress calculations must be consistent across reports.
Section pass analytics must use snapshot-based results.
Review acknowledgement records must be retained.
Randomization analytics must be traceable.
Availability monitoring must reflect current assessment versions.

A4.21 ACCEPTANCE CRITERIA
Assigned assessments must be visible to users.
Assignment reports must be accurate.
Progress metrics must match runtime calculations.
Section failure analytics must be reproducible.
Review acknowledgement reporting must function correctly.
Availability monitoring must identify upcoming expirations.
Notification workflows must operate correctly.
Governance reports must remain auditable.
