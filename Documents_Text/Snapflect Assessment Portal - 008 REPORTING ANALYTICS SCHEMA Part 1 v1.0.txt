REPORT_DEFINITIONS
Purpose
Represents logical report definitions.
Report content is stored in version records.

CREATETABLE report_definitions(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    report_code VARCHAR(100) NOTNULL,    report_name VARCHAR(255) NOTNULL,    ownership_type VARCHAR(50)NOTNULLDEFAULT'System',    report_category VARCHAR(50)NOTNULL,    is_system_report TINYINT(1)NOTNULLDEFAULT0,    status VARCHAR(50)NOTNULLDEFAULT'Draft',    created_by BIGINT UNSIGNED NOTNULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),    modified_by BIGINT UNSIGNED NULL,    modified_date DATETIME(3) NULL,PRIMARYKEY (id),CONSTRAINT ux_report_definitions_uuidUNIQUE (uuid),CONSTRAINT ux_report_definitions_codeUNIQUE (report_code))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

PARAMETER GOVERNANCE
Example:
{"required":["startDate","endDate"]}
Stored in:

EFFECTIVE DATING GOVERNANCE
Report versions become active when:
current_timestamp >= effective_from
and
current_timestamp < effective_to
when effective_to exists.
