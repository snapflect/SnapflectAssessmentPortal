1. PURPOSE
The Security Schema establishes the foundational identity, authentication, authorization, and account management capabilities of the Snapflect Assessment Portal.
This module is the root dependency for all other database modules.
The schema supports:
User Registration
User Authentication
Role-Based Access Control (RBAC)
Session Management
Password Reset
Email Verification
User Profile Management
Security Auditing Support

5. PRIMARY KEY STRATEGY
All tables shall use:
id BIGINT UNSIGNED AUTO_INCREMENT PRIMARYKEY
Characteristics:
Clustered Index
Internal Identifier
Never Exposed Publicly

6. UUID STRATEGY
Every table shall contain:
uuid CHAR(36) NOTNULL
Constraint:
UNIQUE(uuid)
Purpose:
API references
URL references
External integrations
Public identifiers
Internal numeric IDs must never be exposed externally.
