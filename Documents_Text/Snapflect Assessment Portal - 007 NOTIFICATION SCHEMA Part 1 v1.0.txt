NOTIFICATION_TEMPLATES
Purpose
Represents logical notification definitions.
Templates are versioned through:
notification_template_versions
Template content is never stored in the master table.

CREATETABLE notification_templates(id BIGINT UNSIGNED NOTNULL AUTO_INCREMENT,    uuid CHAR(36) NOTNULL,    template_code VARCHAR(100) NOTNULL,    template_name VARCHAR(255) NOTNULL,    ownership_type VARCHAR(50)NOTNULLDEFAULT'System',    notification_type VARCHAR(50)NOTNULL,    is_system_template TINYINT(1)NOTNULLDEFAULT0,    status VARCHAR(50)NOTNULLDEFAULT'Draft',    created_by BIGINT UNSIGNED NOTNULL,    created_date DATETIME(3)NOTNULLDEFAULTCURRENT_TIMESTAMP(3),    modified_by BIGINT UNSIGNED NULL,    modified_date DATETIME(3) NULL,    is_active TINYINT(1)NOTNULLDEFAULT1,PRIMARYKEY (id),CONSTRAINT ux_notification_templates_uuidUNIQUE (uuid),CONSTRAINT ux_notification_templates_codeUNIQUE (template_code))ENGINE=InnoDBDEFAULT CHARSET=utf8mb4COLLATE=utf8mb4_unicode_ci;

SYSTEM TEMPLATE GOVERNANCE
Templates marked:
is_system_template = 1
cannot be deleted through standard administration workflows.
Changes require elevated permissions.

EFFECTIVE DATING GOVERNANCE
Template versions become active when:
current_timestamp >= effective_from
and
current_timestamp < effective_to
when effective_to exists.
