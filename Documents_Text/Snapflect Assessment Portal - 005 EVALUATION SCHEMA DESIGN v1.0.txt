1. PURPOSE
The Evaluation module is responsible for converting candidate responses into scored, reviewed, approved, and published assessment results.
This module provides:
Automated Evaluation
Rule-Based Scoring
Manual Review Workflow
AI-Assisted Evaluation
Result Publication
Evaluation Auditability
Historical Reproducibility

2. EVALUATION ARCHITECTURE
Approved Evaluation Lifecycle
Assessment Submitted        ↓Evaluation Job Created        ↓Auto Evaluation        ↓Manual Review (If Required)        ↓AI Evaluation (Future)        ↓Evaluation Result Generated        ↓Result Published

3. DESIGN PRINCIPLES
Version-Safe Evaluation
All evaluations must be reproducible.
Evaluation logic must be versioned.
Evaluation Rule        ↓Evaluation Rule Version
Historical attempts must continue using the rule version active at evaluation time.

Separation of Concerns
Assessment content:
Assessment Definition Schema
Runtime execution:
Assessment Execution Schema
Scoring logic:
Evaluation Schema
These modules remain independently versioned.

6. EVALUATION_RULE_VERSIONS
Purpose
Stores immutable versions of evaluation rules.
Supports historical reproducibility.

Example
Version 1:
Pass Percentage = 60%
Version 2:
Pass Percentage = 70%
Historical evaluations continue using the original version.

Core Attributes
evaluation_rule_idversion_numberversion_statusrule_namerule_definition_jsonmetadata_json

Rule Definition Strategy
Rules are stored as structured JSON.
Example:
{"passPercentage":70,"negativeMarking":0.25}

Benefits
Supports:
Rule FlexibilityFuture Rule TypesAI Scoring Extensions
without schema changes.
