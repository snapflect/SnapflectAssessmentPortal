A5.2 ASSESSMENT ASSIGNMENT ARCHITECTURE
Purpose

Assigned
Mandatory Assigned

Assignment Lifecycle
Assigned

Started
Completed
Expired
Cancelled

Architecture Rules
Assignments shall be independent from attempts.
One assignment may generate multiple attempts.
Assignment history must remain auditable.
Assignment completion shall be derived from attempt completion.

A5.3 ASSESSMENT AVAILABILITY ARCHITECTURE
Assessment Properties
Available From
Available To

Validation Rules
Assessment cannot start before Available From.
Assessment cannot start after Available To.
Existing attempts remain valid after expiration.
Availability changes must be audited.
Availability rules must be version controlled.

A5.4 RANDOMIZATION ARCHITECTURE
Purpose
Support reproducible question selection.

Supported Strategies
No Randomization
Assessment-Level Randomization
Section-Level Randomization
Question Bank-Level Randomization

Architecture Requirements
Randomization occurs once.
Randomization occurs only during assessment start.
Selected questions are persisted in snapshot.
Randomization seed is persisted.
Historical question selection must be reproducible.

Combined Pass

Architecture Rules

Section pass results must be persisted.
Reports must consume persisted results.
