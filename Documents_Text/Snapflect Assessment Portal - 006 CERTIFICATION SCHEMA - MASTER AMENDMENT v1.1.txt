1. PURPOSE
This amendment incorporates all approved findings from:
Certification Schema Master Review v1.0
The review confirmed that the Certification module is production-ready with no critical or high-severity issues.
This amendment introduces enterprise enhancements focused on:
Certificate Verification Governance
Signature Auditability
Issuance Workflow Control
Fraud Analytics Readiness
Certificate Lifecycle Optimization

2. CERTIFICATES ENHANCEMENT
2.1 Verification Token Expiry Support
New Column
verification_token_expiry_date DATETIME(3) NULL

Purpose
Supports controlled certificate sharing and time-limited verification access.

Example Scenarios
Private Certificate SharingPartner Verification PortalsTemporary Verification LinksEmployer Verification Windows

Governance
When NULL:
Verification Link Never Expires
When populated:
Verification Link Expires Automatically

3. CERTIFICATE_SIGNATURES ENHANCEMENT
3.1 Signer Identity Tracking
New Column
signed_by BIGINT UNSIGNED NULL

Purpose
Stores the identity of the signing authority.

Supported Scenarios
Administrator SigningCertification Authority SigningAutomated Signing ServiceDelegated Certificate Issuance

Foreign Key
ALTERTABLE certificate_signaturesADDCONSTRAINT fk_certificate_signatures_signed_byFOREIGNKEY (signed_by)REFERENCES users(id);

Benefits
Supports:
Signature Audit TrailCompliance ValidationTrust Chain VerificationDigital Credential Governance

4. CERTIFICATE_ISSUANCES ENHANCEMENT
4.1 Issuance Status Tracking
New Column
issuance_status VARCHAR(50)NOTNULLDEFAULT'Pending'

Purpose
Tracks certificate issuance workflow state.

Status Values
PendingCompletedFailedCancelled

Benefits
Supports:
Bulk IssuanceRetry ProcessingOperational MonitoringIssuance Failure Recovery

5. CERTIFICATE_VERIFICATIONS ENHANCEMENT
5.1 Risk Analytics Support
New Column
risk_score DECIMAL(5,2) NULL

Purpose
Stores fraud and anomaly detection scoring.

Example
0.00  = No Risk25.00 = Low Risk50.00 = Medium Risk75.00 = High Risk100.00 = Critical Risk

Future Applications
Fraud DetectionVerification Abuse MonitoringBot DetectionAnomaly DetectionEmployer Verification Analytics

6. INDEXING ENHANCEMENTS
Certificates Lifecycle Optimization
New Composite Index
CREATEINDEX ix_certificates_status_expiryON certificates(    certificate_status,    expiry_date);

Purpose
Optimizes:
Certificate Expiry JobsRenewal ProcessingCertificate Status ReportingLifecycle Management

7. GOVERNANCE RULES
Verification Token Governance
When:
verification_token_expiry_date
exists:
Current Timestamp > Expiry Date
verification requests should return:
Expired

Signature Governance
When:
signed_by
is populated:
The signing actor becomes part of the permanent certificate audit trail.

Issuance Governance
When:
issuance_status = Completed
certificate issuance is considered successful.

When:
issuance_status = Failed
certificate publication must not proceed.

Risk Score Governance
Risk scores are advisory.
They must never automatically:
Revoke CertificatesInvalidate CredentialsModify Certificate Status
without explicit business rules.
