PURPOSE This document defines the complete logical database schema for the Assessment Definition Engine. The schema supports: Assessment Authoring Question Bank Management Competency Frameworks Blueprint Management Assessment Versioning Assessment Snapshots Publishing Workflow Multi-Tenant Isolation RBAC Enforcement Auditability This schema does NOT support: Candidate Assessment Delivery Assessment Attempts Evaluation Certification Reporting AI Proctoring Those modules belong to later sprints. 2. GLOBAL DESIGN STANDARDS All tables must inherit Sprint 01 standards. Primary Key Strategy id BIGINT UNSIGNED AUTO_INCREMENT External Identifier uuid CHAR(36) UNIQUE Tenant Isolation organization_id BIGINT UNSIGNED NOT NULL Required on all tenant-owned entities. Audit Columns created_by created_date modified_by modified_date deleted_by deleted_date is_deleted Status Pattern ACTIVE INACTIVE ARCHIVED 3. ASSESSMENT DOMAIN assessment_categories Purpose: Assessment classification. Examples: Technical Behavioral Psychometric Leadership Coding Columns: id uuid organization_id category_code category_name description status audit columns Indexes: organization_id category_code status assessment_types Purpose: Assessment execution style. Examples: Timed Exam Practice Test Survey Interview Assessment Columns: id uuid organization_id type_code type_name description status audit columns assessment_templates Purpose: Reusable assessment starter templates. Columns: id uuid organization_id template_code template_name assessment_category_id assessment_type_id description status audit columns assessments Purpose: Root assessment entity. Columns: id uuid organization_id assessment_code assessment_name assessment_category_id assessment_type_id template_id version_number current_state description estimated_duration_minutes total_marks pass_percentage is_published status audit columns State Values: DRAFT IN_REVIEW PUBLISHED ARCHIVED Indexes: organization_id assessment_code current_state is_published 4. VERSIONING DOMAIN assessment_versions Purpose: Track every assessment version. Columns: id uuid organization_id assessment_id major_version minor_version version_label change_summary parent_version_id published_date status audit columns Examples: 1.0 1.1 2.0 assessment_snapshots Purpose: Immutable published snapshot. CRITICAL TABLE. Published assessments reference snapshots. Columns: id uuid organization_id assessment_id assessment_version_id snapshot_json snapshot_hash published_by published_date status audit columns Rules: Never Updated Never Deleted Immutable Forever 5. QUESTION BANK DOMAIN question_banks Purpose: Container for questions. COMPETENCY DOMAIN competency_groups Columns: id uuid organization_id group_code group_name description status audit columns Examples: Technical Leadership Communication Problem Solving competencies Columns: id uuid organization_id competency_group_id competency_code competency_name description status audit columns Examples: Laravel Development Conflict Resolution System Design question_competencies Purpose: Question-to-Competency mapping. Columns: id uuid question_id competency_id weight_percentage audit columns assessment_competencies Purpose: Assessment-to-Competency mapping. Columns: id uuid assessment_id competency_id target_percentage audit columns 7. BLUEPRINT DOMAIN assessment_blueprints Purpose: Assessment structure definition. Columns: id uuid organization_id assessment_id blueprint_name description status audit columns Relationship: Assessment 1 : 1 Blueprint blueprint_sections Purpose: Logical assessment sections. Columns: id uuid blueprint_id section_name display_order section_duration_minutes section_weight selection_strategy status audit columns Selection Strategy: DIRECT RULE_BASED blueprint_rules Purpose: Dynamic question selection rules. Columns: id uuid blueprint_section_id difficulty_level tag_id competency_id question_count status audit columns Examples: difficulty=HARD tag=PHP count=10 blueprint_section_questions Purpose: Direct question assignment. Columns: id uuid blueprint_section_id question_id display_order audit columns Used only when: selection_strategy = DIRECT 8. PUBLICATION DOMAIN assessment_reviews Purpose: Review workflow. Columns: id uuid assessment_id review_status review_comments reviewed_by reviewed_date audit columns Status: PENDING APPROVED REJECTED assessment_publications Purpose: Publication history. Columns: id uuid assessment_id assessment_version_id assessment_snapshot_id published_by published_date publication_notes audit columns 9.
